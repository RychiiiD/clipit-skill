## 1. 背景与问题陈述

### 1.1 市场现状

口播视频、直播录屏、课程录制等场景中，创作者普遍面临一个问题：**内容冗余**。一段 20 分钟的录制，真正有价值的内容往往只占 60%-70%，其余是跑题、重复、停顿、互动闲聊。

### 1.2 现有方案局限性

| 方案 | 局限 |
|------|------|
| 手动剪辑 | 耗时巨大，一段 20min 视频需 40-60min 手动拖拽 |
| 现有"智能剪口播" | 只做静音检测，机械去除空白停顿，**不理解内容** |
| 外包剪辑 | 成本高（¥50-200/条），沟通成本大 |

### 1.3 问题总结

创作者的痛点：**花大量时间在剪掉废话上，而非生产好内容**。需要一个真正理解视频内容的工具，能自动识别并裁剪与主题无关的片段。

---

## 2. 产品目标

### 2.1 产品定位

clipit 是一个**跨平台（不限 Agent、不限操作系统）的语义级视频粗剪工具包**。用户只说自然语言——从安装到使用，不敲任何命令——即可完成基于主题理解的视频粗剪。

### 2.2 核心价值主张

> **从"花 1 小时剪视频"变成"说一句话等几分钟"**

### 2.3 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|--------|---------|
| 相关保留率 | ≥95% | 抽样人工审核 |
| 无关裁剪率 | ≥90% | 抽样人工审核 |
| 误剪率 | ≤5% | 用户确认环节统计 |
| 单次处理时长 | ≤1/3 视频时长 | 计时 |
| 用户修改率 | ≤20% | 用户调整参数后重跑的比例 |

## 3. 用户画像与场景

### 3.1 目标用户

| 用户类型 | 典型行为 | 日均视频处理量 |
|---------|---------|-------------|
| 个人创作者 | 录制口播后去废话、去跑题 | 1-3 条 |
| 直播主播 | 从直播回放中切片出主题内容 | 3-10 条 |
| 课程制作人 | 从长录音中提取核心知识点 | 1-2 条长视频 |
| 自媒体运营 | 批量粗剪后分发多渠道 | 5-20 条 |

### 3.2 典型场景

**场景一：个人创作者去废话**

> 小明录了一段 25 分钟的"远程求职技巧"口播，中间聊嗨了讲了 5 分钟自己养猫的事、结尾号召关注重复了 3 遍。他只想保留讲求职技巧的核心内容。

**场景二：直播切片（单主题）**

> 小红 2 小时直播回放，中间回答了弹幕问题、抽了奖、喊了关注。她想把讲"AI 副业"那段单独切出来发抖音。

**场景三：直播切片（多主题）**

> 小刚 1.5 小时直播回放，既讲了"搞钱要站在生产端"又讲了"马斯克第一性原理"。说一句话同时提取两个主题，每个主题输出一条独立短视频。

## 4. 管线流程

clipit 粗剪管线共 6 步，Agent 平台按 SKILL.md 编排执行：

```
Step 1 — 意图提取（NL → video_path + topic + intensity）
Step 2 — 转字幕（Whisper → transcript.json）
Step 2.5 — 字幕清洗（去填充词/重复/卡顿 → transcript_clean.json）
Step 3 — 语义分析（LLM 从"写一篇口播"角度做 keep/cut → decisions.json）
Step 3.5 — 硬规则验证（代码强制修正，AI 不可覆盖）
Step 4 — 用户确认（展示裁剪预览，确认/调参/取消）
Step 5 — 执行拼接（ffmpeg 按 decisions 拼接输出）
Step 6 — 展示报告（时长对比 + 输出路径 + 下一步）
```

多主题场景：Step 3~5 对每个主题循环执行（字幕只转一次），顺序输出每个主题的视频。

## 5. CLI 命令

| 命令 | 功能 |
|------|------|
| `clipit check` | 环境自检（Python/ffmpeg/whisper/PATH） |
| `clipit install` | 自动安装 ffmpeg（三级降级：包管理器 → sudo → 下载静态二进制） |
| `clipit transcribe <video> --model small -o <json>` | 视频转字幕 |
| `clipit clean <json> -o <json>` | 清洗字幕文本 |
| `clipit validate <json> -i <intensity>` | 硬规则验证决策 |
| `clipit splice <video> -d <json> -o <mp4> [--reorder]` | 按决策拼接视频 |

所有命令输出 JSON，Agent 平台解析后展示给用户。

## 8. 边界与约束

### 8.1 在范围内

- 语义级视频粗剪（含字幕提取、字幕清洗、语义分析、验证修正、拼接输出）
- 不限 Agent 平台（Claude Code、Cursor、ChatGPT、自定义 Agent 等均可集成）
- 全自然语言交互（安装、使用、参数调整）
- 自动环境安装与依赖管理

### 8.2 不在范围内

| 功能 | 排除原因 |
|------|---------|
| 视频特效/转场/调色 | 定位是粗剪工具，不做精剪 |
| 字幕美化/花字添加 | 可扩展但不在 v1 |
| 音频降噪/音量均衡 | 由 ffmpeg 预处理，不在核心逻辑内 |
| 画面裁切/比例转换 | 后续可扩展（竖屏适配） |
| 多轨道剪辑 | 超出粗剪范畴 |
| 自动文案生成/改写 | 与主题裁剪无关 |
| 云端处理 / SaaS 服务 | 仅限本地处理 |
| API Key 管理 | 不属于 clipit 职责 |

### 8.3 外部约束

| 约束项 | 说明 |
|--------|------|
| Python 生态 | 依赖 PyPI，使用 openai-whisper、ffmpeg-python |
| ffmpeg | 需安装在系统中，clipit install 自动安装（三级降级兜底） |
| whisper 模型 | 首次使用需下载（~500MB-1.5GB），后续可离线运行 |
| LLM 语义分析 | 语义分析由 Agent 平台调用自身 LLM，不受 clipit 控制 |
| Agent 平台能力 | clipit 管道的完整体验取决于 Agent 平台的 NL 理解、shell 执行和文件系统访问能力 |

## 9. 交互设计（纯自然语言）

### 9.1 安装

```
用户: "帮我装一下 clipit"
Agent: → 检测 Python → pip install → clipit check → clipit install (如需要) → 完成
        → "安装好了，说'帮我粗剪这个视频'开始使用"
```

### 9.2 使用（单主题）

```
用户: "帮我粗剪这个视频，主题是远程求职技巧" + 拖入文件
Agent: → 提取参数 → transcribe → clean → LLM 语义分析
       → validate → 展示预览 → 用户确认
       → splice → 输出视频 + 报告
```

### 9.3 使用（多主题）

```
用户: "帮我粗剪这个视频，主题一是生产端思维，主题二是第一性原理" + 拖入文件
Agent: → 提取参数 + topics[]
       → transcribe（一次） → clean（一次）
       → 对每个主题：LLM 分析 → validate → 展示预览 → 用户确认
       → 逐个 splice → 汇总报告
```

### 9.4 参数调整

```
用户: "剪严格一点" → intensity → aggressive
用户: "太碎了，保留片段别短于 3 秒" → min_keep → 3.0
用户: "宽松点" → intensity → loose
```

所有调整通过自然语言完成，用户无需记忆具体参数名或命令格式。

### 9.5 流程状态设计

| 环节 | 用户看到的内容 | 需要用户操作 |
|------|--------------|------------|
| 安装 | 安装进度条与状态 | 仅说"帮我装一下" |
| 转字幕 | "正在转字幕..." | 无 |
| 字幕清洗 | "正在清洗字幕..." | 无 |
| 语义分析 | "正在分析内容..." | 无 |
| 预览 | 保留占比、裁剪理由摘要 | 确认 / 调整参数 / 取消 |
| 拼接 | "正在拼接视频..." | 无 |
| 完成 | 剪辑报告 | 可选下一步 |

## 10. 架构概要

```
┌─────────────────────────────────────────────────┐
│ Agent 平台（Claude Code / Cursor / 其他）        │
│  ┌───────────────────────────────────────────┐  │
│  │ SKILL.md — 管线编排（Step 1→6）           │  │
│  │ LLM — Step 3 语义分析（平台自身能力）       │  │
│  └───────────────────────────────────────────┘  │
│                        │ CLI 调用                │
│                        ▼                        │
│  ┌───────────────────────────────────────────┐  │
│  │ clipit core — Python 包                    │  │
│  │  transcribe │ clean │ validate │ splice   │  │
│  │  check │ install                          │  │
│  └───────────────────────────────────────────┘  │
│                        │                        │
│                        ▼                        │
│  ┌────────────┐  ┌──────────┐  ┌────────────┐  │
│  │ ffmpeg     │  │ Whisper  │  │ 本地文件系统 │  │
│  └────────────┘  └──────────┘  └────────────┘  │
└─────────────────────────────────────────────────┘
```

### 10.1 设计原则

- **语义分析去中心化** — clipit 只做音视频处理。语义分析用 Agent 自身的 LLM，不配 API Key。
- **代码大于提示词** — LLM 负责语义判断。R1-R6 代码兜底，AI 不可覆盖。
- **单源真理** — `.agents/skills/clipit/SKILL.md` 是唯一规范源。

## 11. 硬规则（R1-R6）

由 `clipit validate` 强制执行，AI 不可覆盖：

| 规则 | 说明 |
|------|------|
| **R1 开篇定题** | 前 2-20s 需点明主题，过短过长不保；首段 keep < 3s 时连保第二段 |
| **R2 短段合并** | keep 段 < min_keep 自动合并/扩展 |
| **R3 裁剪上限** | 裁剪比例不超过 intensity 阈值，超限按 reason 优先级恢复 |
| **R4 同动作去重** | 相邻 keep/keep 或 cut/cut 自动合并 |
| **R5 Reason 校验** | reason 强制标准化为 5 种取值：内容保留/跑题/例子过长/内容重复/空白停顿 |
| **R6 流畅度防护** | keep 段过多或平均过短时裁孤立岛（<3s 两头切），桥接短间隙（<max_gap） |

### 强度预设

| 预设 | 裁剪上限 | min_keep | keep_max | min_avg_keep | max_gap | 场景 |
|------|---------|----------|----------|-------------|---------|------|
| loose | 50% | 2.0s | 20 | 3.0s | 1.5s | 宽松 |
| medium | 70% | 1.0s | 15 | 4.0s | 2.0s | 一般（默认） |
| strict | 85% | 0.5s | 12 | 5.0s | 2.5s | 精编 |
| aggressive | 92% | 0.3s | 10 | 6.0s | 3.0s | 极少量提取 |

## 12. 未决问题与风险

### 12.1 未决问题

| 问题 | 影响 |
|------|------|
| whisper 模型选型：small / medium / large 哪个最优 | 转字幕速度和准确率的平衡 |
| output 格式是否需要支持竖屏（9:16）适配 | 短视频平台分发需求 |
| 是否需要支持 URL 输入（非本地文件） | 远程视频处理场景 |

### 12.2 风险

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|---------|
| ffmpeg 自动安装在某些环境失败 | 中 | 高 | 三级降级（包管理器 → sudo → 静态二进制） |
| whisper 模型下载慢 | 中 | 中 | 显示进度，首次安装时异步下载 |
| Agent 平台 LLM 理解偏差导致误剪 | 低 | 中 | 预览确认兜底 + R1-R6 代码强制修正 |
| 非 Agent Skills 兼容平台无法自动编排 | 中 | 高 | README 明确标注兼容性前提 |
