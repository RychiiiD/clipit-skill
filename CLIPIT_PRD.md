## 1. 背景与问题陈述

### 1.1 市场现状

口播视频、直播录屏、课程录制等场景中，创作者普遍面临一个问题：**内容冗余**。一段 20 分钟的录制，真正有价值的内容往往只占 60%-70%，其余是跑题、重复、停顿、互动闲聊。

### 1.2 现有方案局限性

| 方案 | 局限 |
|------|------|
| 手动剪辑 | 耗时巨大，一段 20min 视频需 40-60min 手动拖拽 |
| 现有"智能剪口播" | 只做静音检测，机械去除空白停顿，**不理解内容** |
| 外包剪辑 | 成本高（¥50-200/条），沟通成本大 |

### 1.3 问题总结

创作者的痛点：**花大量时间在剪掉废话上，而非生产好内容**。需要一个真正理解视频内容的工具，能自动识别并裁剪与主题无关的片段。

## 2. 产品目标

### 2.1 产品定位

clipit 是一个**跨平台（不限 Agent、不限操作系统）的语义级视频粗剪工具包**。用户只说自然语言——从安装到使用，不敲任何命令——即可完成基于主题理解的视频粗剪。

### 2.2 核心价值主张

> **从"花 1 小时剪视频"变成"说一句话等 2 分钟"**

### 2.3 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|--------|---------|
| 相关保留率 | ≥95% | 抽样人工审核 |
| 无关裁剪率 | ≥90% | 抽样人工审核 |
| 误剪率 | ≤5% | 用户确认环节统计 |
| 单次处理时长 | ≤1/3 视频时长 | 计时 |
| 用户修改率 | ≤20% | 用户调整参数后重跑的比例 |

## 3. 用户画像与场景

### 3.1 目标用户

| 用户类型 | 典型行为 | 日均视频处理量 |
|---------|---------|-------------|
| 个人创作者 | 录制口播后去废话、去跑题 | 1-3 条 |
| 直播主播 | 从直播回放中切片出主题内容 | 3-10 条 |
| 课程制作人 | 从长录音中提取核心知识点 | 1-2 条长视频 |
| 自媒体运营 | 批量粗剪后分发多渠道 | 5-20 条 |

### 3.2 典型场景

**场景一：个人创作者去废话**

> 小明录了一段 25 分钟的"远程求职技巧"口播，中间聊嗨了讲了 5 分钟自己养猫的事、结尾号召关注重复了 3 遍。他只想保留讲求职技巧的核心内容。

**场景二：直播切片**

> 小红 2 小时直播回放，中间回答了弹幕问题、抽了奖、喊了关注。她想把讲"AI 副业"那段单独切出来发抖音。

**场景三：批量处理**

> 小丽每天产出 10 条课程视频，统一按"高考数学压轴题"主题裁剪废话和停顿。

## 4. 功能需求

### 4.1 需求列表

| 编号 | 模块 | 描述 | 优先级 |
|------|------|------|--------|
| R001 | 安装 | 用户通过自然语言触发自动安装（检测 Python → pip install → 安装 ffmpeg → 下载 whisper 模型） | P0 |
| R002 | 安装 | 安装过程无需用户手动执行任何命令 | P0 |
| R003 | 安装 | 自动检测操作系统并选择对应的 ffmpeg 安装方式 | P1 |
| R004 | 更新 | 用户通过自然语言"更新 clipit"触发更新 | P1 |
| R005 | 转字幕 | 接收本地视频文件（mp4/mov/avi/mkv），提取音频（16kHz mono） | P0 |
| R006 | 转字幕 | 调用 whisper 将音频转为带时间戳的字幕 JSON | P0 |
| R007 | 转字幕 | 自动选择 whisper 模型（有 GPU→medium，无 GPU→small） | P1 |
| R008 | 语义分析 | Agent 平台使用自身 LLM 分析字幕与主题的相关度，逐句判断保留/裁剪 | P0 |
| R009 | 语义分析 | 支持三级剪辑强度：宽松 / 中等 / 严格 | P0 |
| R010 | 语义分析 | 用户可通过自然语言"剪严格一点"调整强度，无需参数记忆 | P0 |
| R011 | 语义分析 | 支持最小保留片段时长配置，避免输出碎片化 | P1 |
| R012 | 语义分析 | 支持裁剪点前后过渡缓冲（保留 0.3-0.5s 上下文） | P2 |
| R013 | 确认 | 执行裁剪前向用户展示预览（保留占比、裁剪理由摘要） | P0 |
| R014 | 确认 | 每个裁剪决策附带理由（跑题/重复/停顿） | P0 |
| R015 | 确认 | 用户可拒绝并调整参数后重新分析 | P0 |
| R016 | 拼接 | 按决策列表将保留片段拼接为输出视频 | P0 |
| R017 | 拼接 | 支持无损拼接（-c copy）和重编码拼接（时间戳不连续时） | P1 |
| R018 | 报告 | 输出完成后展示剪辑报告（原始时长、保留时长、裁剪内容概览） | P0 |
| R019 | 平台 | 核心层可 pip 安装，与 Agent 平台零耦合 | P0 |
| R020 | 平台 | 提供标准化的 CLI 接口（check / transcribe / splice），输出 JSON | P0 |

### 4.2 需求优先级定义

| 级别 | 含义 |
|------|------|
| P0 | 不可缺失，v1.0 必须交付 |
| P1 | 重要但可延期到 v1.1 |
| P2 | 体验优化，v1.2 后考虑 |
| P3 | 非核心，长期规划 |

## 5. 非功能需求

| 编号 | 类别 | 要求 |
|------|------|------|
| N001 | 性能 | whisper 转字幕（small 模型，20min 视频，有 GPU）≤ 2min |
| N002 | 性能 | 视频拼接时间 ≤ 视频时长的 1/10 |
| N003 | 性能 | 语义分析响应 ≤ 10s（取决于 Agent 平台 LLM） |
| N004 | 兼容性 | 支持主流操作系统（Windows、macOS、Linux 等） |
| N005 | 兼容性 | 支持 Python 3.9+ |
| N006 | 兼容性 | 支持的视频格式：mp4、mov、avi、mkv（ffmpeg 可处理的格式） |
| N007 | 兼容性 | 输入视频大小上限 2GB |
| N008 | 兼容性 | 输入视频时长范围 30s ~ 3h |
| N009 | 音频 | 默认支持中文普通话，whisper 支持的语言均可扩展 |
| N010 | 可用性 | 用户全程无需输入任何命令，仅使用自然语言交互 |
| N011 | 可用性 | 安装失败时给出明确的原因定位和修复指引 |
| N012 | 可靠性 | 非正常退出（中断/崩溃）不应损坏原始视频文件 |
| N013 | 可靠性 | 临时文件在任务结束后自动清理 |
| N014 | 安全性 | clipit core 不管理、不存储任何 API Key |

## 6. 边界与约束

### 6.1 在范围内

- 语义级视频粗剪（含字幕提取、语义分析、拼接输出）
- 不限 Agent 平台（Claude Code、Cursor、ChatGPT、自定义 Agent 等均可集成）
- 全自然语言交互（安装、使用、参数调整）
- 自动环境安装与依赖管理

### 6.2 不在范围内

| 功能 | 排除原因 |
|------|---------|
| 视频特效/转场/调色 | 定位是粗剪工具，不做精剪 |
| 字幕美化/花字添加 | 可扩展但不在 v1 |
| 音频降噪/音量均衡 | 由 ffmpeg 预处理，不在核心逻辑内 |
| 画面裁切/比例转换 | 后续可扩展（竖屏适配） |
| 多轨道剪辑 | 超出粗剪范畴 |
| 自动文案生成/改写 | 与主题裁剪无关 |
| 云端处理 / SaaS 服务 | 仅限本地处理 |
| API Key 管理 | 不属于 clipit 职责 |

### 6.3 外部约束

| 约束项 | 说明 |
|--------|------|
| Python 生态 | 依赖 PyPI，使用 openai-whisper、ffmpeg-python |
| ffmpeg | 需安装在系统中，clipit 自动安装 |
| whisper 模型 | 首次使用需下载（~500MB-1.5GB），后续可离线运行 |
| LLM API | 语义分析由 Agent 平台调用自身 LLM，不受 clipit 控制 |
| Agent 平台能力 | clipit 管道的完整体验取决于 Agent 平台的 NL 理解和展示能力 |

## 7. 交互设计（纯自然语言）

### 7.1 安装

```
用户: "帮我装一下 clipit"
Agent: → 检测 Python → pip install clipit → 安装 ffmpeg → 完成
        → "安装好了，说'帮我粗剪这个视频'开始使用"
```

```
用户: "更新 clipit"
Agent: → pip install --upgrade clipit
```

### 7.2 使用

```
用户: "帮我粗剪这个视频，主题是远程求职技巧" + 拖入文件
Agent: → 提取参数 → 调 transcribe 转字幕 → 调自身 LLM 分析
       → 展示预览确认
       → 确认后调 splice 拼接 → 输出视频 + 报告
```

### 7.3 参数调整

```
用户: "剪严格一点"
用户: "太碎了，保留片段别短于 3 秒"
用户: "保留片段前后各留半秒过渡"
```

所有调整通过自然语言完成，用户无需记忆具体参数名或命令格式。

### 7.4 流程状态设计

| 环节 | 用户看到的内容 | 需要用户操作 |
|------|--------------|------------|
| 安装 | 安装进度条与状态 | 仅说"帮我装一下" |
| 转字幕 | "正在转字幕..." | 无 |
| 语义分析 | "正在分析内容..." | 无 |
| 预览 | 保留占比、裁剪理由摘要 | 确认 / 调整参数 / 取消 |
| 拼接 | "正在拼接视频..." | 无 |
| 完成 | 剪辑报告 | 可选下一步 |

## 8. 未决问题与风险

### 8.1 未决问题

| 问题 | 需要决策方 | 影响 |
|------|-----------|------|
| whisper 模型选型：small / medium / large 哪个作为默认值 | 产品 + 工程 | 影响转字幕速度和准确率的平衡 |
| 是否需要支持 URL 输入（非本地文件） | 产品 | P1 需求，推迟到 v1.1 决定 |
| 输出视频是否保留原始画质，还是统一压缩 | 产品 | 影响文件大小和拼接速度 |
| 是否需要支持暂停/继续（中断恢复） | 产品 | 长视频场景是否需要 |

### 8.2 风险

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|---------|
| ffmpeg 自动安装在某些环境失败 | 中 | 高 | 提供手动安装引导 |
| whisper 模型下载慢 | 中 | 中 | 显示进度，首次安装时异步下载 |
| Agent 平台 LLM 理解偏差导致误剪 | 低 | 中 | 预览确认兜底，强度可调 |
| 用户输入格式不兼容 | 低 | 中 | check 时检测，明确告知 |
