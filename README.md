## 前置条件

| 条件 | 说明 |
|------|------|
| **Python 3.9+** | clipit core 和 Whisper 需要 Python 运行环境 |
| **ffmpeg** | 视频拼接依赖 ffmpeg（`clipit install` 可自动安装） |
| **Shell 执行能力** | Agent 平台必须能执行 shell 命令 |
| **本地文件访问** | Agent 平台必须能读写本地文件 |

语义分析由 Agent 平台自身的 LLM 完成，**不需要配置任何 API Key**。

## 为什么不一样

市面上大多数"智能剪辑"只做**静音检测**——机械地去掉空白停顿，完全不理解内容在说什么。

clipit 的差异在于三层结构：

| 层级 | 做什么 | 谁负责 |
|------|--------|--------|
| **语义理解** | 理解视频在说什么，判断哪些内容围绕主题、哪些跑题 | LLM |
| **硬规则兜底** | 开篇必须点题、短段自动合并、裁剪不超上限——代码强制修正，AI 不可覆盖 | clipit validate |
| **音视频处理** | 转字幕、清洗填充词、拼接输出 | clipit core |

简单说：**LLM 负责"好不好"，代码负责"对不对"**。AI 判断错了，规则拉回来。

### 其他差异点

- **成文导向** — 不是逐句裁剪，而是从"写一篇完整口播"的角度做精编，确保输出有开头、有论证、有案例、有结论
- **多主题提取** — 一条直播回放，每个主题独立跑一遍分析，各自剪出一条完整的口播短视频。主题可以相近（如从不同角度讲同一概念），也可以完全不相干（如从同一场直播中提取"搞钱"和"创业"两个话题）。实际效果取决于源视频是否有足够的内容分别支撑每个主题
- **零 API Key** — 不绑定任何 LLM 服务商，用 Agent 平台自身的能力
- **跨平台** — 基于 Agent Skills 协议，Claude Code、Cursor 等 50+ runtime 可用

## 工作原理

| 步骤 | 谁做 |
|------|------|
| 转字幕 | Whisper 提取字幕 |
| 字幕清洗 | 去填充词、重复、卡顿 |
| 语义分析 | **LLM** — 从"写一篇口播"的角度精编，多主题逐个分析 |
| 验证修正 | 代码强制规则兜底，AI 不可覆盖 |
| 用户确认 | 展示预览，你确认后再执行 |
| 拼接输出 | ffmpeg 拼接成最终视频 |

## 工具边界

clipit 只做**语义粗剪**，产出的是"素材半成品"。精剪加工留给你熟悉的剪辑工具：

| 不负责 | 说明 |
|--------|------|
| 精剪/微调 | 片段级别的拖拽调参 |
| 转场/特效 | 画面过渡、滤镜 |
| 花字/字幕美化 | 字幕样式、动画 |
| 调色/美颜 | 色彩校正、人像美化 |
| 音频处理 | 降噪、背景音乐 |
| 多轨道剪辑 | 画中画、分屏 |
